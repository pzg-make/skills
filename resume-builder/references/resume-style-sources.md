# Resume Style Sources

Use this reference when choosing or adapting a resume visual style.

## Default External Source

Use GitHub repository `mmmlllnnn/ResumeCollection` as the default style/template source:

- Repository: https://github.com/mmmlllnnn/ResumeCollection
- Purpose: resume templates for job search and education, including Chinese and English resumes.
- Observed categories: Chinese resumes, English resumes, multi-page resumes, resume covers, and simple table resumes.
- License signal: GitHub shows the repository as MIT licensed; verify the license before copying or redistributing files.

## Selection Rules

Choose style by use case:

- Chinese job search: start from the Chinese resume category; prefer clean one-page layouts for most business, product, operations, finance, education, and healthcare roles.
- English applications: start from the English resume category; prefer ATS-readable single-column or restrained two-column layouts.
- Senior or research profiles: consider multi-page references only when the user has enough relevant experience, publications, projects, or academic history.
- Cover-based submissions: use resume-cover references only when the target market or school/employer expects a cover page.
- Data-entry or HR forms: use simple table resume references when the user requests a form-like resume or the receiving organization expects tabular information.

## Adaptation Rules

- Use the repository to identify layout patterns, color direction, section composition, and density.
- Do not change an imported resume's existing style unless the user explicitly asks for style/template changes.
- Do not copy a template verbatim when a tailored design is more appropriate; adapt spacing, typography, and hierarchy to the user's content.
- Prefer ATS-friendly variants for online applications; avoid decorative templates when machine parsing matters.
- Keep generated PDF text selectable and keep DOCX/HTML/Markdown editable sources.
- When downloading a template, preserve the original file separately and work on a copy.

## Practical Search Hints

When browsing the repository, look for:

- Role keywords such as product manager, engineer, developer, finance, marketing, operations, teacher, nurse, designer, or student.
- Style keywords such as simple, blue, green, two-column, table, cover, fresh, business, creative, or compact.
- File compatibility notes; the repository mentions renaming suffixes between `.docx` and `.doc` if compatibility issues occur.
